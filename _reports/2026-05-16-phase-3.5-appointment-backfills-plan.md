## 10. KoyaOS dogfood lessons (carved from Phase 3 → Phase 3.5 split)

- **Don't bundle multiple invariant paths into one phase.** Phase 3's original draft carried an optional Section 5 that turned the invariant assertion into a branch (+0 or +5 on appointments). Codex verifier flagged it; carving 3.5 out gave each phase a single clean delta vector. KoyaOS pattern: one phase = one invariant path.
- **Evidence tiers belong in separate phases.** Strong-evidence (precise date) appointments shipped in one phase; date-ambiguous ones held for Sam-readback. Mixing tiers in a single transaction forces the operator to gate on the weakest link.
- **Sam-readback is a phase input, not a verification step.** The 3 deferred rows in §3 are blocked on Sam, not on Codex or Cowork. KoyaOS pattern: any phase that depends on operator clarification names the operator and the specific artifact (which card, which question) explicitly in the plan.
